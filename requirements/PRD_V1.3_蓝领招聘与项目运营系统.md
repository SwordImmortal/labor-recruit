# 蓝领招聘与项目运营系统 PRD（V1.3）

## 1. 文档信息
- 产品名称：蓝领招聘与项目运营系统（RPO/BPO）
- 版本：V1.3
- 文档状态：评审版
- 目标用户：招聘专员、招聘主管、管理员

## 2. 背景与目标
### 2.1 业务背景
公司承接客户招聘委托（RPO/BPO），需要同时管理招聘过程、入离职过程、项目履约与月度结算。

### 2.2 产品目标
1. 提升招聘团队管理效率（任务清晰、过程可管、结果可追）。
2. 统一维护候选人全流程状态，沉淀可复用人才库。
3. 支撑项目持续供人、入离职管理和服务费结算。

## 3. 范围与边界
### 3.1 本期范围（MVP）
- 客户管理
- 项目管理（招聘状态 + 运营状态）
- 岗位管理
- 线索池与分配
- 候选人管理
- 人才库管理
- 入离职管理
- 渠道管理
- 结算管理
- 报表中心（日报/周报/月报）
- 系统管理（用户/角色/菜单/权限/字典/数据权限）

### 3.2 暂不包含
- 外部系统对接（电话/短信/OA/薪资等）
- 渠道成本与ROI分析

## 4. 角色与权限
### 4.1 招聘专员
- 管理本人线索与候选人
- 更新流程状态与跟进记录
- 查看本人日报/周报/月报

### 4.2 招聘主管
- 查看团队数据
- 执行线索分配与改派
- 处理离职后的“是否补招”决策
- 查看团队报表

### 4.3 管理员
- 全量数据权限
- 系统配置（字典、流程、权限）
- 用户与角色管理

## 5. 核心业务流程
### 5.1 招聘主流程
线索 -> 交换电话微信 -> 约面 -> 到面 -> 入职

### 5.2 运营主流程
入职 -> 在职 -> 离职

### 5.3 项目子流程（按项目配置）
示例：京东家政启用“购包 -> 退包”。

## 6. 项目状态流转规则（双状态）
### 6.1 状态定义
- 招聘状态：待启动 / 招聘中 / 已招满 / 补招中 / 停招
- 运营状态：待履约 / 服务中 / 暂停服务 / 已终止

### 6.2 关键口径
净在岗人数 = 累计入职人数 - 累计离职人数

### 6.3 流转规则
1. 待启动 -> 招聘中：主管/管理员启动招聘。
2. 招聘中 -> 已招满：净在岗人数 >= 项目目标人数。
3. 已招满 + 服务中可并行存在：招满后项目仍可继续履约与结算。
4. 离职后触发提醒，不自动补招。
5. 主管选择“需要补招”后，状态改为补招中并生成补招任务。
6. 主管选择“暂不补招”时，状态不变，必须记录原因。
7. 离职后24小时未决策，系统二次提醒（仍不自动补招）。

## 7. 功能模块需求
### 7.1 客户管理
#### 页面
- 客户列表
- 客户详情
- 联系人管理
- 合同管理

#### 核心字段
- 客户名称、行业、合作城市、负责人
- 联系人信息（姓名/电话/微信）
- 合同编号、起止日期、结算周期、备注

#### 核心规则
- 项目必须归属客户
- 客户状态：潜在 / 合作中 / 暂停 / 终止

### 7.2 项目管理
#### 页面
- 项目列表
- 项目详情
- 项目流程模板

#### 核心字段
- 客户、项目名称、业务类型（RPO/BPO）
- 城市、岗位、目标人数、负责人
- 开始日期、结束日期、招聘状态、运营状态

#### 周期用途
- 定义招聘窗口
- 用于预警与报表统计
- 用于结项判断

### 7.3 岗位管理
#### 设计说明
- 系统岗位字典：统一岗位标准名称（如保洁、酒店服务员）
- 项目岗位：项目下可多选岗位，并配置各岗位需求人数
- 候选人岗位：当前应聘岗位单选（从项目岗位中选择）

### 7.4 线索池与分配
#### 线索池用途
- 统一承接各渠道新增线索
- 判重后进入“待分配”

#### 分配方式
- 手动分配
- 自动轮询
- 按地区/岗位分配

#### 规则
- 支持超时未跟进回收并重分
- 支持无效线索标记（需填写原因）

### 7.5 候选人管理
#### 核心字段
- 基础：姓名、年龄、手机号、微信号、备注
- 归因：渠道来源、渠道账号、邀约人、跟进招聘人、应聘项目、应聘岗位
- 面试：面试时间、面试备注、面试反馈
- 结果：是否到面、是否入职、入职项目、跟进时间和进度

#### 规则
- 约面状态必须填写面试时间
- 面试结束必须填写面试备注与面试反馈
- 状态变更必须记录操作人、时间、备注

### 7.6 人才库管理
#### 准入规则
- 有手机号或微信号即可入库（OR规则）

#### 判重规则
- 手机号优先判重，其次微信号
- 命中任一则合并到同一候选人

#### 功能
- 人才筛选
- 激活重分配
- 历史跟进追踪

### 7.7 入离职管理
#### 页面
- 入职登记
- 在职列表
- 离职登记

#### 核心字段
- 入职项目、姓名、手机号、身份证号
- 购包日期、入职日期、线索来源、入职城市、线索归属人、备注
- 离职日期、离职原因

#### 规则
- 手机号/身份证号按文本存储
- 身份证号默认脱敏展示
- 离职后可回流人才库

### 7.8 渠道管理
#### 功能
- 渠道维护（Boss/快手/58/抖音/线下）
- 渠道账号维护
- 启用/停用

### 7.9 结算管理
#### 页面
- 结算单列表
- 结算单详情
- 开票管理
- 回款管理

#### 状态
待生成 -> 待确认 -> 待开票 -> 待回款 -> 部分回款/已回款

#### 核心字段
- 结算月、客户、项目、应结人数、单价、应收金额
- 开票状态、回款状态、已收/未收金额、备注

#### 规则
- 运营状态为“服务中”的项目参与月结
- 金额调整需填写原因并记录审计日志

### 7.10 报表中心
#### 报表周期
- 日报 / 周报 / 月报

#### 统计维度
- 专员维度
- 团队维度
- 项目维度

#### 指标
- 线索量
- 交换电话微信量
- 约面量
- 到面量
- 入职量
- 净在岗人数
- 补招次数
- 应收/已收/未收金额

## 8. 项目专项规则（京东家政）
1. 必须“已购包 + 购包日期”才可入职。
2. 退包状态禁止入职，需重新购包后再入职。

## 9. 系统管理（统一字典 + RBAC）
### 9.1 字典管理
#### 字典范围
- 渠道、岗位、城市
- 招聘状态、运营状态
- 项目子状态（购包/退包）
- 面试反馈、未到面原因、离职原因
- 结算状态

#### 字典规则
- 字典项支持：启用/停用、排序、默认值、适用范围（全局/项目）
- 被业务引用的字典项不可删除，仅可停用
- 停用仅影响新数据，不影响历史数据回显

### 9.2 用户管理
- 用户新增/编辑/启停
- 角色绑定
- 密码重置
- 登录日志查看

### 9.3 角色管理
- 角色新增/编辑/复制
- 菜单权限、按钮权限、API权限配置

### 9.4 菜单管理
- 菜单树配置
- 路由与显示控制

### 9.5 权限与数据权限
- 权限维度：页面、按钮、API
- 数据范围：本人 / 团队 / 按项目 / 按客户 / 全量
- 多角色用户按并集生效

### 9.6 审计日志
- 记录关键配置与业务变更（操作人、时间、前后值）

## 10. 关键业务规则汇总
1. 人才库准入：手机号或微信号即可入库。
2. 判重：手机号优先，微信号其次。
3. 京东家政：购包前置，退包不可入职。
4. 离职后先提醒，确认补招后才进入补招中。
5. 24小时未决策补招，系统再次提醒。

## 11. MVP验收标准
1. 候选人全流程可追溯至入离职，关键操作有日志。
2. 双状态模型可稳定运行，离职补招规则符合“确认后触发”。
3. 京东家政购包前置规则拦截生效。
4. 人才库准入与判重规则生效。
5. 日报/周报/月报口径与业务明细一致。
6. 结算流程形成“生成-确认-开票-回款”闭环。
7. 系统管理（用户/角色/菜单/权限/字典）可配置并生效。
