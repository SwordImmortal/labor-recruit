## 数据模型

### 核心实体
- **User** - 用户（管理员/主管/专员）
- **Candidate** - 候选人（线索 → 加微信 → 约面 → 到面 → 入职/流失）
- **Project** - 项目（RPO/BPO，双状态模型）
- **Onboarding** - 入职记录
- **Resignation** - 离职记录
- **Channel** - 渠道来源
- **DictType/DictItem** - 字典管理

### 业务流程
```
招聘流程：线索 → 加微信 → 约面 → 到面 → 入职
运营流程：入职 → 在职 → 离职
项目状态：招聘状态 + 运营状态（双状态模型）
```
