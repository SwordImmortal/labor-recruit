## 一、项目概述

### 1.1 基本信息

| 项目 | 信息 |
|------|------|
| **项目名称** | 蓝领招聘与项目运营系统 |
| **项目代号** | labor-recruit |
| **业务模式** | RPO（招聘流程外包）/ BPO（业务流程外包） |
| **目标用户** | 招聘专员、招聘主管、管理员 |

### 1.2 业务背景

公司承接客户招聘委托，需要同时管理：
- 招聘过程：线索 → 面试 → 入职
- 运营过程：入职 → 在职 → 离职
- 项目履约：供人目标 → 月度结算

### 1.3 系统目标

1. **提升效率** - 任务清晰、过程可管、结果可追
2. **人才沉淀** - 候选人全流程状态管理，可复用人才库
3. **支撑交付** - 项目供人、入离职管理、服务费结算

## 三、核心业务流程

### 3.1 招聘主流程

```
线索 → 加微信 → 约面 → 到面 → 入职
```

### 3.2 运营主流程

```
入职 → 在职 → 离职
```

### 3.3 项目状态（双状态模型）

**招聘状态：**
- 待启动 → 招聘中 → 已招满 → 补招中 → 停招

**运营状态：**
- 待履约 → 服务中 → 暂停服务 → 已终止

### 3.4 关键口径

```
净在岗人数 = 累计入职人数 - 累计离职人数
```

## 五、特殊业务规则

### 5.1 京东家政项目

```
购包前置规则：
1. 必须先购包才能入职
2. 退包状态禁止入职
3. 需重新购包后才能再入职
```

### 5.2 离职补招流程

```
1. 员工离职 → 系统提醒主管
2. 主管决策：
   - 需要补招 → 状态改为"补招中" → 生成补招任务
   - 暂不补招 → 状态不变 → 必须填写原因
3. 24小时未决策 → 系统二次提醒
```

### 5.3 线索归属

```
三种角色：
- 邀约人：线索最初是谁拉来的
- 跟进人：当前谁在跟进
- 归属人：入职后结算归属

归属变更记录日志，可追溯
```
